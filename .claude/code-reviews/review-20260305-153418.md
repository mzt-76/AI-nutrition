### Issue #1

**Severity:** low
**Category:** Logic
**File:** `frontend/src/pages/MealPlanView.tsx:78-80`
**Verified:** ⚠️ probable

**Issue:**
`fetchPlan` is defined inside the component body but used as a `useEffect` dependency and an `onClick` handler. React's exhaustive-deps lint rule (if enabled) would flag this as a missing dependency since `fetchPlan` is recreated every render.

**Detail:**
Currently works correctly because:
- `session` is stable (useState reference), `id` is stable (URL param)
- The effect deps `[id, session]` correctly capture the actual variables that trigger re-fetch
- `fetchPlan` is not in the dep array (the effect calls it inline)

This is a common React pattern for "extract function for reuse as onClick + useEffect body". Not a bug but slightly fragile — if eslint `react-hooks/exhaustive-deps` is strict, it may warn.

**Current Code:**
```tsx
const fetchPlan = async () => { ... };
useEffect(() => { fetchPlan(); }, [id, session]);
```

**Suggested Fix:**
Acceptable as-is. If strictness desired, wrap with `useCallback`:
```tsx
const fetchPlan = useCallback(async () => { ... }, [id, session]);
useEffect(() => { fetchPlan(); }, [fetchPlan]);
```
