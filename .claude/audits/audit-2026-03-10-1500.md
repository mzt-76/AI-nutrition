### High

#### H1: Missing sanitize_user_text() on API query input [Agent B]
- **file**: `src/api.py:269-278, 1365`
- **category**: security
- **what**: `AgentRequest.query` validates only `max_length=5000` but never calls `sanitize_user_text()`. Query is passed directly to agent at line 1365 without sanitization. The sanitization function exists and is used in skill scripts but not at the API boundary.
- **why**: Prompt injection vector. Defense-in-depth gap at the main entry point.
- **suggested_fix**: Import `sanitize_user_text` in api.py, apply to `request.query` before passing to agent.
- **confidence**: probable

#### H2: UUID validation missing on user_id in multiple endpoints [Agent B]
- **file**: `src/api.py:368-384, 631, 741, 878`
- **category**: security
- **what**: `user_id: str` query parameters in `list_conversations`, `meal-plans`, `daily-log`, `favorites` endpoints lack UUID format validation. `_validate_uuid()` exists but is only used for entry_id/plan_id.
- **why**: Business logic bugs if user_id formats are inconsistent. Defense-in-depth issue.
- **suggested_fix**: Apply `_validate_uuid(user_id)` in all endpoints that accept user_id.
- **confidence**: probable

#### H3: MIME type validation too permissive (prefix matching) [Agent B]
- **file**: `src/api.py:406, 448-450`
- **category**: security
- **what**: Uses `startswith("image/")` allowing any `image/*` type including `image/x-executable`. Should be exact match whitelist.
- **why**: Executable files could be disguised as images and uploaded.
- **suggested_fix**: Change to exact match set: `{"image/jpeg", "image/png", "image/gif", "image/webp", "text/plain", "application/pdf"}`.
- **confidence**: probable

#### H4: agent.py implicit Optional on tool parameters [Agent A + D merged]
- **file**: `src/agent.py:309-326`
- **category**: correctness
- **what**: `update_my_profile` has `age: int = None`, `gender: str = None` etc. — PEP 484 requires explicit `int | None = None`. Mypy flags 8 errors here. Additionally, no validation prevents all-None calls (silent no-op).
- **why**: Weakens type safety, mypy errors block CI, users get no feedback on empty updates.
- **suggested_fix**: Change all params to `T | None = None`. Add early return if all params are None.
- **confidence**: verified

#### H5: RecalculateRequest.gender is str instead of Literal [Agent A + B merged]
- **file**: `src/api.py:330, 1268`
- **category**: correctness
- **what**: `gender: str` in RecalculateRequest vs `Literal["male", "female"]` expected by `mifflin_st_jeor_bmr()`. Defensive check at line 1254 is redundant if Pydantic validates.
- **why**: Runtime type errors if unexpected gender values pass validation.
- **suggested_fix**: Change to `gender: Literal["male", "female"]`. Remove redundant check at line 1254.
- **confidence**: verified

#### H6: `any` type in ComponentRenderer COMPONENT_CATALOG [Agent C]
- **file**: `frontend/src/components/generative-ui/ComponentRenderer.tsx:14`
- **category**: standards
- **what**: `Record<string, React.FC<any>>` with eslint-disable comment. Violates rule 2 (zero tolerance for `any`).
- **why**: Defeats Zod validation purpose. Runtime errors from incorrect props won't be caught by TypeScript.
- **suggested_fix**: Create union type of all valid component props, use `Record<string, React.FC<ValidProps>>`.
- **confidence**: verified

#### H7: skill-creator in _VALID_SKILLS but directory doesn't exist [Agent D]
- **file**: `src/agent.py:108-118`, `evals/test_skill_loading.py:32-41`
- **category**: standards
- **what**: `_VALID_SKILLS` frozenset includes "skill-creator" but `skills/skill-creator/` doesn't exist (it's in `.claude/skills/`). Eval EXPECTED_SKILLS also references it.
- **why**: Runtime FileNotFoundError if agent tries to load the skill.
- **suggested_fix**: Remove "skill-creator" from `_VALID_SKILLS` and `EXPECTED_SKILLS` in evals.
- **confidence**: verified

### Medium

#### M1: Service key potentially exposed in httpx logs [Agent B]
- **file**: `src/api.py:211-223`
- **category**: security
- **what**: Supabase service key passed in Authorization header per-request. If httpx logger is DEBUG or exceptions capture headers, key appears in logs.
- **suggested_fix**: Store key in client default headers, add logging filter to redact "apikey" headers.
- **confidence**: probable

#### M2: CORS falls back to localhost in production [Agent B]
- **file**: `src/api.py:154-178`
- **category**: security
- **what**: If `CORS_ORIGINS` env var is missing in production, code logs warning but falls back to localhost origins instead of failing.
- **suggested_fix**: Raise RuntimeError if CORS_ORIGINS missing in production. Only allow localhost fallback in development.
- **confidence**: probable

#### M3: Uninitialized _http_client race condition [Agent B]
- **file**: `src/api.py:215-217`
- **category**: security
- **what**: `verify_token()` checks `_http_client is None` but can be called before lifespan initializes, causing RuntimeError instead of graceful 401.
- **suggested_fix**: Ensure all global clients initialized before endpoints are callable.
- **confidence**: probable

#### M4: Async functions using sync Supabase client [Agent A]
- **file**: `src/db_utils.py:22-273`
- **category**: efficiency
- **what**: Multiple `async def` functions call synchronous `supabase.table(...).execute()`, blocking the event loop.

- **confidence**: probable

#### M6: useMessageHandling hook is 333 lines with 13 parameters [Agent C]
